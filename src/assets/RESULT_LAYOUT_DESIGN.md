# 흑백개발자: 결과 화면 UI/UX 개편 명세서

## 1. 개요 (Overview)
기존의 중앙 정렬(Center Aligned) 방식은 정보가 순차적으로 떠서 다소 정적인 느낌을 줌.
이를 **좌우 분할(Split Layout)** 로 변경하여, 

시각적 정보(요리 이미지)와 텍스트 정보(심사평)를 동시에, 그러나 역동적으로 보여줌으로써 '대결 결과'의 긴장감을 극대화함.

## 2. 레이아웃 구조 (Layout Structure)
- **Container**: Flexbox를 사용하여 화면을 좌우로 나눔. (모바일에서는 상하로 자동 배치 `flex-wrap`)

- **Left Column (Visual)**:
  - 핵심 요소: AI가 생성한 고퀄리티 요리 결과물 이미지.
  - 역할: 시선을 사로잡는 'Kick'.
- **Right Column (Information)**:
  - 핵심 요소: 심사평(Verdict), 등급(Tier), 다시하기 버튼.
  - 역할: 사용자가 읽어야 할 정보 전달.

## 3. 애니메이션 전략 (Animation Strategy)
Framer Motion을 활용하여 양쪽에서 중앙으로 모이는 연출을 사용.

| 영역 | 애니메이션 효과 | 상세 설정 |
| :--- | :--- | :--- |
| **Left (이미지)** | **Slide In form Left** | 화면 왼쪽 밖(`x: -100`)에서 중앙으로 부드럽게 진입. |
| **Right (텍스트)** | **Fade In from Right** | 화면 오른쪽(`x: 50`)에서 투명도가 서서히 걷히며 등장. |
| **Background** | **Blur & Scale** | 배경에 결과 이미지를 깔고 블러 처리하여 공간감을 줌. |

## 4. 기대 효과
- 영화 포스터나 대전 격투 게임의 결과 화면 같은 고급스러운 느낌.
- 텍스트 가독성 확보 (이미지와 텍스트 영역 분리).
