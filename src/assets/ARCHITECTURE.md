# 🏗️ Architecture & Design Decisions (설계 및 구현 의도)

## 1. 프로젝트 개요 (Project Overview)
본 프로젝트 **'흑백개발자(Code Class War)'**는 넷플릭스 예능 '흑백요리사'의 컨셉을 차용하여, 사용자의 코드를 AI가 심사하고 계급(Tier)을 매겨주는 **코드 리뷰 서비스**입니다.

단순한 텍스트 기반의 코드 리뷰를 넘어, **"사용자 경험(UX)과 몰입감"**을 극대화하기 위해 프론트엔드 인터랙션과 시각적 피드백에 중점을 두고 설계했습니다.

---

## 2. 기술 스택 선정 이유 (Tech Stack & Rationale)

### ⚡ Core: React + Vite
- **선정 이유:** 해커톤 및 단기 프로젝트 특성상 **빠른 초기 세팅과 빌드 속도**가 필수적이었습니다. CRA(Create React App) 대비 압도적으로 빠른 Vite를 선택하여 개발 생산성을 높였습니다.

### 🎨 Styling: Tailwind CSS
- **선정 이유:** 별도의 CSS 파일을 관리하는 오버헤드를 줄이고, 클래스명 고민 없이 **직관적이고 빠르게 UI를 구축**하기 위함입니다.
- **활용:** `flex`, `grid` 시스템을 활용한 반응형 레이아웃과, 다크 모드 기반의 디자인 시스템을 신속하게 적용했습니다.

### 🎬 Animation: Framer Motion
- **선정 이유:** 이 서비스의 핵심은 **'심사 결과가 나올 때의 긴장감'**입니다. 단순 CSS Transition으로는 구현하기 힘든 **페이지 간 전환(Page Transition), 순차적 등장(Staggering), 숫자 카운팅** 등의 고품질 애니메이션을 구현하기 위해 채택했습니다.
- **주요 구현:** `AnimatePresence`를 활용한 자연스러운 화면 전환, 결과 화면의 슬라이드 인 효과.

### 🧩 Icons: Lucide React
- **선정 이유:** 일관된 라인 스타일의 아이콘을 통해 UI의 통일성을 유지하고, 가벼운 용량으로 성능 저하를 방지했습니다.

## 3. UI/UX 디자인 철학 (Design Philosophy)

### 3.1. 몰입형 스토리텔링 (Immersive Storytelling)
사용자가 단순히 기능을 쓰는 것이 아니라 **"참가자가 되어 심사를 받는 느낌"**을 주도록 설계했습니다.
- **Landing:** 웅장한 타이틀과 배경으로 긴장감 조성.
- **Loading:** 단순히 '로딩 중'이 아닌, **"코드를 맛보는 중", "가비지 컬렉터와 대화 중"** 같은 위트 있는 멘트와 사운드, GIF를 활용해 대기 시간을 엔터테인먼트로 승화했습니다.

### 3.2. 정보의 위계 (Visual Hierarchy)
결과 화면(`ResultPage`)에서 정보 과부하를 막기 위해 **좌우 분할 레이아웃**을 채택했습니다.
- **좌측 (감성):** 등급(Tier) 이미지와 거대한 텍스트로 **직관적인 결과**를 먼저 제시.
- **우측 (이성):** 심사평, 상세 스탯, 개선점 등 **논리적인 데이터**를 사용자가 차분히 읽을 수 있도록 배치.

---

## 4. 디렉토리 구조 (Directory Structure)

유지보수성과 확장성을 고려하여 **관심사 분리(Separation of Concerns)** 원칙에 따라 구조화했습니다.
