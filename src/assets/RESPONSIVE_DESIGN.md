# 흑백개발자: 반응형 레이아웃 개선 (Responsive Fix)

## 1. 문제점 (Problem)
- **현상**: 결과 화면(Result)에서 내용이 많거나 화면이 작은 경우(모바일), 하단 버튼이나 텍스트가 잘려서 보이지 않음.
- **원인**: 
  1. 최상위 컨테이너에 `height: 100vh` (화면 꽉 채움)와 `overflow: hidden` (넘치면 자름)이 설정되어 있음.
  2. 내부 콘텐츠가 화면보다 길어질 경우 스크롤이 불가능함.

## 2. 해결 전략 (Solution)
- **스크롤 허용 (Enable Scroll)**: 
  - `overflow: hidden`을  **`overflow-y: auto`** 로 변경하여, 세로 내용이 넘치면 스크롤이 생기도록 함.
- **최소 높이 설정 (Min-Height)**:
  - 컨텐츠 영역을 강제로 `height: 100%`로 고정하지 않고, **`min-height: 100vh`**로 설정.
  - 내용이 적을 땐 화면에 꽉 차고, 내용이 많으면 알아서 늘어나게 함.
- **여백 확보 (Padding)**:
  - 모바일에서 위아래 여백(`padding: 4rem 1rem`)을 충분히 주어 답답하지 않게 함.

## 3. 적용 결과
- **PC**: 화면이 크므로 한눈에 다 보임 (기존과 동일).
- **Mobile**: 내용이 세로로 길게 배치되며, 자연스럽게 스크롤해서 끝까지 볼 수 있음.
