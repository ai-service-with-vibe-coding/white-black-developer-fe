# 흑백개발자: 목업 및 API 연동 가이드

### 1. 설계 의도 (Why so?)
* **좌우 2분할 레이아웃**: 결과 이미지(감성)와 데이터 보고서(이성)를 분리하여 사용자에게 '재미'와 '정보'를 동시에 전달합니다.
* **단계별 애니메이션**: 
    * 1단계: 왼쪽에서 이미지가 슬라이드인 되어 등급을 먼저 각인시킵니다.
    * 2단계: 1초 뒤 보고서(스테이터스)가 페이드인 되어 상세 분석 결과를 읽게 유도합니다. (시선 처리 유도)
* **고밀도 로딩 화면**: 분석 시간이 길기 때문에(1분), 무한 반복되는 GIF와 진행 문구를 배치하여 이탈률을 방지합니다.

### 2. 주요 UI 컴포넌트
* **StatusRow**: `Lucide-React` 아이콘과 `Framer-motion` 게이지 바를 결합하여 전문적인 분석 리포트 느낌을 구현했습니다.
* **Backdrop Blur**: 보고서 배경에 블러 처리를 하여 고급스러운 다크 모드 UI를 완성했습니다.

### 3. 추후 API 연동 시 주의사항
* **Timeout**: 브라우저 기본 Timeout 설정이 1분보다 짧을 수 있으므로, `axios` 사용 시 `timeout: 120000` 설정을 권장합니다.
* **Data Mapping**: 백엔드에서 오는 점수 객체의 키값(`security`, `quality` 등)이 현재 프론트엔드의 인터페이스와 일치하는지 확인하십시오.
* **Error Handling**: 분석 중 서버 에러 발생 시 사용자를 다시 `INPUT` 단계로 되돌려 코드를 수정할 수 있게 해야 합니다.
